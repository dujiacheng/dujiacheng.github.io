gitblit的仓库备份

Veröffentlicht am 2016-09-26

1.首先修改 gitblit的配置文件,把federation.passphrase 的值改为1

➜ data git:(master) pwd
/usr/local/apache-tomcat-7.0.72/webapps/gitblit-1.8.0/WEB-INF/data
➜ data git:(master) vi defaults.properties

1598 # RESTART REQUIRED (only to enable or disable federation)
1599 federation.passphrase = 1

2.重启tomcat, 然后在tomcat/logs/catalina.out中搜索token , 能找到如下信息

2016-09-26 11:38:25 [INFO ] Federation ALL token = 39be117190334fa206f474a7c8fd27bab4405141
2016-09-26 11:38:25 [INFO ] Federation USERS_AND_REPOSITORIES token = 7ecb704e9376732d11cf1ae99de0ec52f07751ea
2016-09-26 11:38:25 [INFO ] Federation REPOSITORIES token = 975be3e3fd91fc8956016ceeba21c92ad5cf98b8

3.gitblit官网下载fedclient, 解压

http://dl.bintray.com/gitblit/releases/fedclient-1.8.0.zip
我是直接在Downloads文件夹中解压了
➜ fedclient-1.8.0 pwd
/Users/du/Downloads/fedclient-1.8.0
➜ fedclient-1.8.0 ls
LICENSE ext federation.properties
NOTICE fedclient.jar users.conf

4. 修改fedclient的配置

➜ fedclient-1.8.0 vi federation.properties

federation.example1.url = http://localhost:8080/gitblit-1.8.0/
federation.example1.token = 39be117190334fa206f474a7c8fd27bab4405141
federation.example1.frequency = 120 mins
federation.example1.folder =
federation.example1.bare = true
federation.example1.mirror = true
federation.example1.mergeAccounts = true

url是tomcat的地址
token 则是catalina.out 中找到的, 如果是备份所有仓库用ALL token
其他信息不变

5. 备份

java -jar fedclient.jar

terminal中没有err 出现就是正常.会多出一个git文件夹,包含所有仓库
➜ fedclient-1.8.0 ls
LICENSE fedclient.jar users.conf
NOTICE federation.properties
ext git
