1.0 ISO/OSI七层模型


从服务器A发送到 服务器B
A是左边从上往下打包发送. B是从下往上一层层解包

物理层

设备之间的比特流的传输,物理接口,电气特性等

数据链路层

成帧, 用MAC地址访问媒介, 错误检测与修正

网络层

提供逻辑地址(源ip,目的ip), 选路

传输层

选择TCP 还是UDP, 可靠与不可靠的传输,传输前的错误检测,流控

会话层

对应用会话管理, 同步()

表示层

数据的表现形式(比如UTF-8编码的转换,或者mp3格式 ),特定功能的实现如- 加密

应用层

提供用户接口

应用层 -> 打开浏览器 输入地址打开网址, 点击发送邮件
表示层 -> 进行各种编码的翻译
会话层 -> 是本地保存, 还是网络传输
传输层-> 写清楚什么协议
网络层-> 提供逻辑地址(源ip,目的ip邮箱地址), 选路
数据链路层-> mac地址

1.1 TCP/IP 四层模型

与OSI有什么区别? 看图

OSI更像是模型车, TCP是原型车, TCP参照了OSI来制造

网络接口层

网络接入层与OSI参考模型中的物理层和数据链路层相对应.
它负责监视数据在主机和网络之间的交换. 事实上,TCP/IP
本身并未定义该层的协议, 而由参与互连的各网络使用自己的
物理层和数据链路层协议, 然后与TCP/IP的网络接入层进行
连接. 地址解析

网际互联层

网际互联层对应于OSI参考模型的网络层, 主要解决主机到
主机的通信问题. 它所包含的设计数据包在整个网路上的逻
辑传输.
该层有三个主要协议:
网际协议(ip),
互联网组管理
协议(IGMP)和互联网控制报文协议(ICMP)

传输层

传输层对应于OSI的传输层, 为应用层尸体提供端到端
的通信功能, 保证了数据包的顺序传送及数据的完整性.
该层定义了两个主要的协议: 传输控制协议(TCP)和用户
数据报协议(UDP)

1.2 IP地址

00000000.00000000.00000000.00000000
到
11111111.11111111.11111111.11111111
的是ipv4协议可以提供的ip地址
换算成 10进制就是
0.0.0.0 到 255.255.255.255
但也不是所有范围内的ip都可以使用, 具体看下图分类

私有ip可以有效的节约公网ip的使用

私有ip通过和公有ip进行转换, 来连接互联网.

A类通过第一组数字来表示 是否一个网段,后三组代表同网段的不同主机

1.0.0.0 和 2.0.0.0 已经是两个网段了. 要通信的话, 需要通过路由器.
同一网络通信只需要交换机.
A类网络 有 2^24-2 个地址可以分配, 为什么要-2 呢?
因为 1.0.0.0 代表这个网络本身, 不能进行分配
1.255.255.255 代表这个网络的广播地址, 也不能进行分配
以此类推, 2.0.0.0 和 2.255.255.255

最大主机数, 代表每个网络可分配的地址.
也就是 00000001.00000000.00000000.00000000 ~ 00000001.11111111.11111111.11111111
除开00000001之后的3个8位,也就是24位,就是可调配的地址区间

B类通过前两组数字来表示是否一个网段,后两组代表同网段的不同主机

C类通过前三组数字来表示是否一个网段

子网掩码

###ip属于哪个网段是拿子网掩码来确定的, ip是不能决定的.但是每个ip都有一个标准子网掩码.以上划分都是按照, 标准子网掩码来定义的.
举例证明

IP Address 192.168.0.1

Subnet Mask 255.255.255.0

转换为二进制后进行 按位与计算

IP Address 11010000.10101000.00000000.00000001

Subnet Mask 11111111.11111111.11111111.00000000

AND 运算后
11000000.10101000.00000000.00000000

转换成 十进制
192.168.0.0

IP Address 192.168.0.254

Subnet Mask 255.255.255.0

AND 运算后 转换成10进制
也是 192.168.0.0

所以验证了,ip和子网掩码一起看 才可以确认该ip的网段

2.1 Linux配置IP地址的方法

如果有人在局域网内搭建了DHCP服务器, 那我就可以点击自动获取IP地址(window下图形界面为例)

Linux配置IP地址的方法

1) ifconfig命令临时配置IP地址
ifconfig eth0 192.168.0.200 netmask 255.255.255.0 (eth0代表一块网卡)

2 setup工具永久配置IP地址(redhat专有)

$ setup

修改后需要运行一条命令

$ service network restart

3 修改网络配置文件

$ vi /etc/sysconfig/network-scripts/ifcfg-echo0


BOOTPROTO=none 是否自动获取IP(none, static 都代表手动获取ip),如果是dhcp 那只需要手动填写蓝色字体部分就可以.

主机名配置

$ vi /etc/sysconfig/network
NETWORKING=yes #网络是否开启/工作
HOSTNAME=localhost.localdomain #主机名

DNS配置文件

vi /ect/.conf

nameserver 202.106.0.20 #名字服务器 dns
search localhost # 比如网站是 imooc.com , 结果我只输入了imooc, 那就会进行 search imooc.localhost

2.3 虚拟机网络参数配置

1.配置LinuxIP地址

$ setup

2.启动网卡

$ vi /etc/sysconfig/network-scripts/ifcfg-etho0
把 ONBOOT=no 改为 ONBOOT=yes
service network restart
重启网络服务

3.修改UUID

vi /ect/sysconfig/network-scripts/ifcfg-eth0 #删除MAC地址行

rm -rf /ect/udev/rule.d/70-persistent-net.rules #删除网卡和MAC绑定文件 , 然后重启系统

4.设置虚拟机网络连接方式

在windows下设置虚拟机的网络连接, 推荐使用桥接(这样虚拟机和局域网内的虚拟机可以通信, 虚拟机和外网也可以同喜)

5.修改桥接网卡

选择虚拟机桥接物理机的哪一块网卡
点击编辑 -> 虚拟网络编辑器 -> 已桥接到 然后选择有线/无线
wireless 是无线网卡

#网络查看的命令

##关闭/启用 网卡

  • ifdown 网卡设备名 #禁用这块 网卡

  • ifup 网卡设备名 #启用该网卡设备

##网络状态查看命令
lsof -i tcp:port （port替换成端口号，比如6379）可以查看该端口被什么程序占用，并显示PID，方便KILL

  • netstat 选项
    选项:
    -t: 列出TCP协议端口
    -u: 列出UDP协议端口
    -n: 不使用域名与服务名
    -l: 仅列出在监听状态网络服务
    -a: 列出所有的网络连接

    $netstat -an # 自己编写的socket程序也会被显示出来

    [root@rpi2 ~]# netstat -an | grep ESTABLISHED #连接到本机的ip有哪些
    [root@rpi2 ~]# netstat -an | grep ESTABLISHED | wc -l #连接到本机的ip数量

$netstat -rn # 显示网关 与 $ route -n 等价
